\section{Конвертованные данные}\label{FMT.EnvelopedData}

Формат конвертованных данных задается типом~\code{EnvelopedData}, который
определен в СТБ~34.101.23. 

Контейнер~\code{EnvelopedData} заполняется по правилам СТБ~34.101.23
со следующими уточнениями.

\begin{enumerate}
\item
Версия синтаксиса (компонент~\code{version}) должна равняться~$0$. 

\item
Опциональные компоненты~\code{originatorInfo} и 
\code{unprotectedAttrs} должны быть опущены. 

\item
Идентификатор алгоритмов шифрования 
(компонент \code{contentEncryptionAlgorithm}, вложенный  
в~\code{encryptedContentInfo}) должен быть выбран из перечня, 
заданного в~\ref{CRYPTO.Encr}.

\item
Тип конвертуемых данных (компонент~\code{eContentType}, вложенный 
в~\code{encryptedContentInfo}) должен принимать значение~\code{id-data}. 
Перед конвертованием подписанных данных они должны быть вложены в контейнер 
\code{EncapsulatedContentInfo} (определен в СТБ~34.101.23), причем компонент 
\code{eContentType} этого контейнера должен принимать значение 
\code{id-signedData}.

\item
Список сведений о получателях (компонент~\code{recipientInfos})
должен содержать единственный контейнер~\code{RecipientInfo}, 
и в этом контейнере должен быть выбран компонент~\code{ktri} 
типа~\code{KeyTransRecipientInfo}. 

Компонент~\code{ktri} должен формироваться следующим образом:
\begin{enumerate}
\item
версия применяемого синтаксиса (компонент~\code{version})
должна равняться~$0$;
\item
идентификатор получателя (\code{rid}) должен задаваться через выбор 
варианта~\code{issuerAndSerialNumber};
\item
в~\code{keyEncryptionAlgortithm} должен быть установлен идентификатор
алгоритмов~\code{bign-keytransport}, описанных в~\ref{CRYPTO.Transport}.
\end{enumerate}
\end{enumerate}
