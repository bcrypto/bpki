\section{Запрос и ответ службы заверения данных}\label{FMT.DVCS}

\subsection{Формат запроса}\label{FMT.DVCS.Req}

СЗД должна поддерживать сервис проверки действительности ЭД (vsd) и не должна
поддерживать другие сервисы.

Формат запроса СЗД задается типом~\code{DVCSRequest}, который определен в 
СТБ~34.101.81. Запрос содержит общие данные запроса и заверяемый ЭД.

Общие данные запроса указываются в компоненте~\code{requestInformation}
типа~\code{DVCSRequestInformation}. Тип описывает версию применяемого синтаксиса
и запрашиваемый клиентом сервис. Должны быть выбраны версия~1 и сервис vsd. Все
опциональные компоненты~\code{DVCSRequestInformation} должны быть опущены.

Заверяемый ЭД кодируется строкой октетов и указывается в компоненте
\code{message}, вложенном в компонент~\code{data} типа~\code{DVCSRequest}.

\subsection{Формат ответа}\label{FMT.DVCS.Resp}

Формат ответа СЗД задается контейнером типа~\code{SignedData}. Контейнер должен
формироваться по правилам, заданным в~\ref{FMT.SignedData}.
 
Непосредственно подписывается значение типа~\code{DVCSResponse}. Это значение
инкапсулируется в~\code{SignedData} с
идентификатором~\code{id-ct-DVCSResponseData}.

СЗД должна выносить один из трех вердиктов:
\code{granted}~--- документ признан действительным,
\code{waiting}~--- проверка не завершена,
\code{rejection}~--- документ признан недействительным.
%
При первых двух вердиктах в~\code{DVCSResponce} должен быть выбран
компонент~\code{dvCertInfo}, в случае третьего вердикта~---
компонент~\code{dvErrorNote}.

В компоненте~\code{dvCertInfo} должны быть опущены все вложенные компоненты,
кроме \code{version}, \code{dvReqInfo}, \code{messageImprint},
\code{serialNumber}, \code{responseTime}.
%
Вложенный компонент~\code{dvStatus} должен присутствовать в случае
вердикта~\code{waiting} и должен быть опущен при вердикте~\code{granted}.
%
В \code{responseTime} должна быть выбрана опция \code{genTime}, т.~е. время
должно задаваться значением типа \code{GeneralizedTime}.

В компоненте~\code{dvErrorNote} должен быть опущен вложенный
компонент~\code{transactionIdentifier}.
	
Для компонентов~\code{dvCertInfo} должны соблюдаться следующие правила:
\begin{enumerate}
\item
Компонент \code{version} должен принимать значение 1.
\item
В~\code{dvReqInfo} должна быть перенесена общая часть запроса. 
\item
Алгоритм хэширования, указанный в~\code{messageImprint},
должен соответствовать алгоритму ЭЦП, который используется для 
подписи ответа.
\end{enumerate}
