\section{Ответ удостоверяющего центра}\label{FMT.BPKIResp}

Формат ответа УЦ на запрос другой стороны определяется следующим типом~АСН.1:
\begin{verbatim}
BPKIResp ::= SEQUENCE { 
  statusInfo  PKIStatusInfo,
  requestId   OCTET STRING(SIZE(32)),
  nonce       OCTET STRING(SIZE(8)) OPTIONAL }
\end{verbatim}

Компонент \code{statusInfo} содержит статус обработки запроса. Тип
\code{PKIStatusInfo} определен в СТБ~34.101.82.

В случае успеха вложенные в~\code{statusInfo} компоненты~\code{statusString}
и~\code{failInfo} должны быть опущены, а вложенный компонент~\code{status}
должен принимать значение~\code{granted}.
%
Если ошибка произошла по причине загруженности УЦ или потому что обработка
запроса требует времени, то~\code{statusString} и~\code{failInfo} также должны
быть опущены, а~\code{status} должен принимать значение~\code{waiting}.
%
При других ошибках в~\code{status} должно быть выбрано
значение~\code{rejection}, в~\code{failInfo} должен быть указан один из кодов
ошибки, определенных в СТБ~34.101.82, а в~\code{statusString} ошибка может быть
дополнительно прокомментирована.

Компонент \code{requestId} содержит идентификатор запроса~--- его хэш-значение,
вычисляемое с помощью алгоритма \code{belt-hash} (см.~\ref{CRYPTO.Hash}).

Опциональный компонент~\code{nonce} содержит синхропосылку~--- случайную строку
октетов, которая делает ответы УЦ неповторяющимися даже при
одинаковых~\code{requestId}. Компонент используется только в ответах на
повторные запросы.
